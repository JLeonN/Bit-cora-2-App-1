### **Módulo de Etiquetas - Bitácora II**

Sistema completo para generar etiquetas con códigos de barras Code 39 en documentos PDF personalizados.

---

### Estado funcional actual (v4.2.19)

- En web se muestra botón específico de descarga PDF y se oculta el botón enviar.
- En móvil se mantiene compartir nativo del PDF generado.
- Se corrigió el flujo para no intentar compartir cuando la salida es descarga web.
- Se optimizó el render PDF para bajar picos de memoria en generación extensa.
- Se corrigió la configuración de plugins Quasar para notificaciones y loading en runtime.

### 📦 **Componentes Principales**

#### **PaginaEtiquetas.vue**
- **Página principal** del módulo de etiquetas
- **Ruta**: `/etiquetas`
- **Header con título** y selector de tamaño
- **Selector de tamaño**: 
  - **10x15 cm** (por defecto)
  - **5x10 cm**
  - **2.5x6.7 cm**
- **FormularioEtiqueta** para agregar etiquetas
- **TablaEtiquetas** con lista de etiquetas
- **Integración con BarraBotonesInferior** para navegación
- **Modal de confirmación** para eliminar etiquetas
- **Persistencia automática** de etiquetas
- **Responsive design** adaptativo
- **Sistema de polling**: Verifica cada 2 segundos si hay nuevas etiquetas desde ubicaciones

#### **FormularioEtiqueta.vue**
- **Formulario completo** para agregar etiquetas
- **Campos**:
  - **Código del artículo**: Input con buscador inteligente integrado
  - **Descripción**: Autocompletado desde base de datos
  - **Ubicación**: Con formateo automático y botón limpiar
  - **Cantidad de copias**: Input numérico (mínimo 1)
- **Botón de cámara** para escanear códigos
- **Integración con CamaraEscaneo.vue** para múltiples escaneos
- **Autocompletado completo**: Al seleccionar artículo del buscador, autocompleta descripción y ubicación
- **Validaciones** con feedback visual y animaciones de error
- **Formateo automático** de ubicaciones (espacios → guiones)
- **Botón IconTrash** para limpiar ubicación cuando cambia de zona

#### **TablaEtiquetas.vue**
- **Lista completa** de etiquetas agregadas
- **Header con contadores**:
  - Etiquetas totales
  - Copias totales
  - Códigos repetidos (si hay)
  - Artículos inexistentes (si hay)
- **Columnas**:
  - **Nombre y Código**: Muestra nombre de la base de datos + código
  - **Ubicación**: Ubicación del artículo
  - **Cantidad**: Control inline con botones +/-
  - **Acciones**: Editar y eliminar
- **Detección de códigos duplicados** (fondo naranja)
- **Detección de artículos inexistentes** (fondo rojo)
- **Controles de cantidad inline**: Incrementar/decrementar sin modal
- **Modal personalizado para confirmar limpieza**: Usa ModalEliminar en lugar de confirm() nativo
- **Botón limpiar todo** con confirmación
- **Responsive** con diseño de cards en móvil
- **Integración con base de datos** para mostrar nombres

#### **ModalEditarEtiqueta.vue**
- **Modal para editar** etiquetas existentes
- **Campos editables**: código, descripción, ubicación, cantidad
- **Validaciones** completas con feedback visual
- **Botones**: guardar y cancelar
- **Integración** con sistema de etiquetas

### 🔗 **Integración con Ubicaciones**

#### **Sistema de Recepción de Etiquetas**
- **Polling cada 2 segundos**: Detecta cambios en almacenamiento
- **Actualización automática**: Lista se actualiza sin intervención del usuario
- **Convivencia de fuentes**: Etiquetas manuales + enviadas desde ubicaciones
- **Sin distinción visual**: Ambas fuentes se muestran igual en la tabla
- **Persistencia compartida**: Usa mismo almacenamiento (`usoAlmacenamientoEtiquetas.js`)

**Datos recibidos desde ubicaciones**:
- `codigo`: Código del artículo
- `descripcion`: Nombre obtenido de base de datos
- `ubicacion`: Ubicación actual (no la antigua)
- `cantidad`: 1 (por defecto)
- `tamano`: 10x15cm (por defecto)
- `id`: Timestamp único

### 📄 **Sistema de Generación de Documentos**

#### **GeneradorEtiquetasPDF.js**
- **Motor principal** para generar documentos PDF
- **Biblioteca**: `jsPDF` para crear archivos PDF
- **Configuración modular**: Usa archivos de configuración externos
- **Dimensiones personalizadas**: Cada configuración define tamaño de página
- **Layout adaptativo**: Ajusta elementos según configuración
- **Detección de entorno**: Navegador web vs app móvil
- **Descarga directa en navegador**: Sin usar Capacitor Filesystem
- **Nombres personalizados**: Integra nombre de usuario desde configuración

**Contenido por etiqueta** (una página por etiqueta):
1. **Código del artículo**: 
   - Posición, fuente y tamaño configurables
   - **Sistema adaptativo**: Tamaño de fuente cambia según longitud del código
   - 4 niveles: corto, mediano, largo, muy largo
2. **Código de barras**: PNG generado por GeneradorCodigoBarra.js
3. **Descripción**: 
   - **Tamaño automático**: Ajuste inteligente según longitud del texto
   - **Multilínea**: División automática de texto largo
   - **Interlineado configurable**
   - **Máximo de líneas configurable**
   - **7 niveles de tamaño**: muy corta, corta, mediana, larga, muy larga, extra larga, gigante
4. **Ubicación**: Posición configurable (por defecto esquina inferior izquierda)

**Características avanzadas**:
- **Salto de página** automático entre etiquetas
- **Respeta cantidad de copias**: Genera páginas duplicadas
- **Guardado temporal** con `@capacitor/filesystem` en `Directory.Cache` (solo móvil)
- **Descarga directa en navegador**: No requiere Capacitor
- **Conversión a base64** para compatibilidad con Capacitor (solo móvil)
- **Retorna**: URI del archivo + nombre del archivo

**Función principal**: `generarDocumentoEtiquetas(listaEtiquetas, configuracion)`

### 📐 **Configuraciones de Etiquetas Disponibles**

El sistema es **totalmente modular** y soporta múltiples tamaños de etiquetas. Cada configuración define propiedades específicas para cada tamaño.

#### **ConfiguracionEtiqueta10x15.js**
- **Tamaño**: 10x15 cm (100mm x 150mm)
- **Orientación**: Horizontal (landscape)
- **Márgenes**: 4mm
- **Código de artículo**: Tamaños adaptativos: 50pt, 40pt, 36pt, 25pt
- **Código de barras**: Dimensiones: 90mm x 17mm, Tamaño interno: 'grande'
- **Descripción**: Tamaños: 60pt → 24pt (7 niveles), Ancho máximo: 140mm, Máximo líneas: 7
- **Ubicación**: 10pt, esquina inferior izquierda

#### **ConfiguracionEtiqueta5x10.js**
- **Tamaño**: 5x10 cm (50mm x 100mm)
- **Orientación**: Horizontal (landscape)
- **Márgenes**: 2.5mm
- **Código de artículo**: Tamaños adaptativos: 25pt, 20pt, 18pt, 12pt
- **Código de barras**: Dimensiones: 73mm x 11mm, Tamaño interno: 'mediano'
- **Descripción**: Tamaños: 30pt → 12pt (7 niveles), Ancho máximo: 93mm, Máximo líneas: 5
- **Ubicación**: 6.5pt, esquina inferior izquierda

#### **ConfiguracionEtiqueta2.5x6.7.js**
- **Tamaño**: 2.5x6.7 cm (25mm x 67mm)
- **Orientación**: Horizontal (landscape)
- **Márgenes**: 1.5mm
- **Código de artículo**: Tamaños adaptativos: 12pt, 10pt, 9pt, 7pt
- **Código de barras**: Dimensiones: 50mm x 6mm, Tamaño interno: 'chico'
- **Descripción**: Tamaños: 15pt → 6pt (7 niveles), Ancho máximo: 62mm, Máximo líneas: 4
- **Ubicación**: 4.5pt, esquina inferior izquierda

**Características comunes de todas las configuraciones**:
- Sistema de tamaños automáticos para código y descripción
- Posiciones Y dinámicas según cantidad de líneas
- Fuente: Helvetica Bold
- Alineación: Center (código y descripción)
- Fondo blanco, sin bordes (configurables para debug)

**Extensibilidad**: Sistema preparado para agregar nuevas configuraciones fácilmente

---

#### **GeneradorCodigoBarra.js**
- **Generación de códigos de barras** en formato PNG base64
- **Biblioteca**: `jsbarcode` para crear códigos Code 39
- **Formato**: Code 39 (compatible con escáneres industriales)

**Tamaños configurables** (uso interno): 
- `grande`: height 80, fontSize 26, width 3
- `mediano`: height 60, fontSize 20, width 2
- `chico`: height 40, fontSize 16, width 2

**Características**:
- **Fondo blanco**, barras negras
- **No muestra texto inferior** (se agrega manual en el documento)
- **Asteriscos automáticos**: Code 39 los agrega automáticamente
- **Conversión a mayúsculas** automática
- **Validación** de caracteres compatibles con Code 39

**Funciones exportadas**:
- `generarCodigoBarraPNG(codigo, tamano)`
- `validarCodigoParaBarra(codigo)`
- `limpiarCodigoParaBarra(codigo)`

**Caracteres válidos Code 39**: A-Z, 0-9, símbolos: `-`, `.`, `$`, `/`, `+`, `%`, espacio

### 💾 **Sistema de Persistencia de Etiquetas**

**Las etiquetas se guardan automáticamente usando `@capacitor/preferences`**

#### **usoAlmacenamientoEtiquetas.js**
- **Ubicación**: `src/components/BaseDeDatos/usoAlmacenamientoEtiquetas.js`
- **Clave de almacenamiento**: `historial_etiquetas`

**Funciones exportadas**:
- `guardarEtiquetas(etiquetas)`
- `obtenerEtiquetas()`
- `eliminarEtiquetas()`

**Persistencia automática**:
- Se guarda al agregar nueva etiqueta
- Se guarda al editar etiqueta existente
- Se guarda al eliminar etiqueta

- Permite continuar trabajo en sesiones posteriores

**Formato de datos guardados**: `{ id, codigo, descripcion, ubicacion, cantidad, tamano }`

### 📁 **Exportación y Compartir**

#### **Nombre de archivo**
- **Formato**: `Etiquetas - [nombre_usuario] - [id_configuracion].pdf`

#### **Proceso de exportación**:
1. Usuario click en botón "Enviar"
2. **Loading** con mensaje "Generando documento PDF..."
3. Se genera documento con `GeneradorEtiquetasPDF.js`
4. **En navegador**: Descarga directa del PDF
5. **En móvil**: Se guarda en `Directory.Cache` con Capacitor y se comparte
6. **Notificación** de éxito o error
7. **Las etiquetas NO se borran** después de generar el PDF

#### **CompartirExcel.js** (reutilizado)
- Mismo módulo usado en Pedidos y Ubicaciones
- **Share API nativa** de Capacitor
- Compatible con PDF
- Solo se usa en móvil

### 📝 **Notas Importantes**

- Las etiquetas **SÍ se persisten** automáticamente en cada operación
- El **tamaño de fuente del código** se ajusta automáticamente según longitud
- El **tamaño de fuente de la descripción** se ajusta automáticamente según longitud
- Los **códigos de barras** son formato Code 39 (estándar industrial)
- El **documento PDF** tiene formato personalizado según configuración seleccionada
- Hay **tres configuraciones disponibles**: 10x15cm, 5x10cm y 2.5x6.7cm
- Cada etiqueta ocupa **una página completa** del documento
- Las **copias** se generan como páginas duplicadas consecutivas
- Los **artículos inexistentes** se pueden agregar (útil para nuevos productos)
- El **sistema de ubicaciones** es opcional - puede dejarse "Sin ubicación"
- El **buscador inteligente** facilita encontrar artículos rápidamente
- La **base de datos Excel** debe estar cargada para autocompletado óptimo
- Los **tamaños del código de barras** (chico/mediano/grande) son configuración interna
- El **sistema es modular**: Fácil agregar nuevas configuraciones de tamaño
- **Funciona en navegador y móvil**: Detección automática de entorno
- El **tamaño seleccionado** se guarda con cada etiqueta individual
- Los **nombres de archivo** incluyen el nombre de usuario configurado
- El **polling cada 2 segundos** permite recibir etiquetas desde ubicaciones sin recargar página
- Las **etiquetas de ambas fuentes** conviven en la misma lista sin distinción visual
